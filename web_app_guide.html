1 라이브러리 불러오기 (import)

import numpy as np import pandas as pd import pickle import re from konlpy.tag import Okt from sklearn.metrics.pairwise import linear_kernel from scipy.io import mmread from flask import Flask, render_template, request, jsonify
import는 "도구를 꺼내 온다"는 뜻이에요. 요리를 하기 전에 칼, 도마, 냄비를 꺼내는 것처럼, 코딩도 필요한 도구(라이브러리)를 먼저 불러와야 해요.
라이브러리하는 일
numpy (np)숫자 계산을 빠르게 해주는 도구. 이 앱에선 보너스 점수 배열(0으로 채워진 긴 줄)을 만들 때 써요.
pandas (pd)엑셀처럼 표(CSV 파일)를 읽고 다루는 도구. 9031개의 피부 데이터를 읽어요.
pickle파이썬 객체를 파일로 저장하거나 불러오는 도구. 미리 학습된 TF-IDF 모델(.pkl 파일)을 불러올 때 써요.
re정규식(Regular Expression) 도구. 한글만 남기고 나머지(영어, 숫자, 특수문자)를 지울 때 써요.
Okt한국어 형태소 분석기. "검버섯이 생겼어요" → ["검버섯", "생기다"] 처럼 단어를 쪼개는 역할을 해요.
linear_kernel두 문장이 얼마나 비슷한지 점수(0~1)를 계산하는 함수예요. (코사인 유사도)
mmreadTF-IDF 행렬 파일(.mtx 형식)을 읽는 도구예요.
Flask 등웹 서버를 만드는 핵심 도구예요. Flask=서버 틀, render_template=HTML 페이지 반환, request=요청 데이터 읽기, jsonify=JSON 형태로 결과 반환.

app = Flask(__name__) # 서버 시작 시 데이터 및 모델 한 번만 로드 print("Loading data and models...") df_skin = pd.read_csv('./datasets/skin_data_final.csv') tfidf_matrix = mmread('./models/Tfidf_skin_data.mtx').tocsr() with open('./models/tfidf.pkl', 'rb') as f: tfidf = pickle.load(f) okt = Okt()

Flask로 웹 서버를 하나 만들어요. __name__은 "이 파일이 있는 폴더를 기준으로 설정해줘"라는 의미예요.
df_skin = pd.read_csv(...)
피부 고민 데이터가 담긴 CSV 파일을 읽어서 표 형태(DataFrame)로 저장해요. 이 표에는 성별, 연령대, 피부 타입, 질문, 답변 등이 9031행으로 들어 있어요.
tfidf_matrix = mmread(...).tocsr()
미리 계산해 둔 "단어 점수 표"를 불러와요. 9031개 답변 각각이 어떤 단어를 얼마나 많이 쓰는지 수치로 변환해 놓은 거예요. .tocsr()은 이 표를 빠르게 계산할 수 있는 형식으로 바꾸는 것이에요.
tfidf = pickle.load(f)
새로운 문장을 같은 방식의 숫자로 바꿔주는 "번역기 모델"을 불러와요. 이걸 써야 사용자 입력도 9031개 답변과 비교할 수 있는 숫자로 바꿀 수 있어요.
데이터·모델을 서버가 시작할 때 딱 한 번 불러오는 이유: 추천 버튼을 누를 때마다 파일을 다시 읽으면 너무 느려요. 마치 도서관에서 책을 매번 서가에서 꺼내는 것보다, 책상 위에 미리 꺼내 두는 게 빠른 것처럼요.

3 불용어 로드

try: df_stopwords = pd.read_csv('../movie_review/datasets/stopwords.csv') stopwords = df_stopwords['stopword'].tolist() except: stopwords = ['아', '휴', '나', '우리', '을', '를', ...]
불용어(stopwords)란 분석에 도움이 되지 않는 단어예요. "아", "나", "을", "를" 같이 의미 없는 단어는 유사도 계산 전에 미리 걸러냅니다.
검색할 때 "나는 여드름이 있어요"를 입력하면, "나는", "이", "있어요"는 중요하지 않고 "여드름"만 중요해요. 불용어 목록은 이런 불필요한 단어들을 모아 놓은 리스트예요.
try / except는 파일이 없어도 앱이 꺼지지 않게 해줘요. 파일이 없으면 기본 불용어 목록을 대신 사용해요.

4 텍스트 전처리 함수 — preprocess_input()

def preprocess_input(text): text = re.sub('[^가-힣]', ' ', text) tokens = okt.pos(text, stem=True) words = [w for w, p in tokens if p in ['Noun', 'Verb', 'Adjective'] and len(w) > 1 and w not in stopwords] return ' '.join(words)
이 함수는 사용자가 입력한 문장을 AI가 이해할 수 있는 "핵심 단어 모음"으로 바꿔 줘요. 마치 독후감에서 핵심 단어만 형광펜으로 표시하는 것처럼요.

단계별 설명

단계코드설명예시
1단계 re.sub('[^가-힣]', ' ', text) 한글이 아닌 모든 글자(영어, 숫자, 특수문자)를 공백으로 바꿔요. "여드름!! oily" → "여드름 "
2단계 okt.pos(text, stem=True) 문장을 단어 단위로 쪼개고, 각 단어의 품사(명사/동사/형용사 등)를 알아내요. stem=True는 단어를 기본형으로 바꿔요. ("생겼어요" → "생기다") "검버섯이 생겼어요" → [("검버섯","Noun"), ("생기다","Verb")]
3단계 if p in ['Noun','Verb','Adjective'] 명사, 동사, 형용사만 남기고 나머지 품사(조사, 어미 등)는 버려요. "을", "이", "는" 같은 조사 제거
4단계 len(w) > 1 한 글자짜리 단어는 의미가 너무 모호해서 버려요. "나", "것" 제거
5단계 w not in stopwords 불용어 목록에 있는 단어도 버려요. "우리", "저희" 제거
6단계 ' '.join(words) 남은 단어들을 공백으로 이어 붙여서 하나의 문자열로 만들어요. ["검버섯","생기다"] → "검버섯 생기다"

@app.route('/') def index(): return render_template('index.html')
@app.route('/')는 "누군가 http://127.0.0.1:5000/ 주소에 접속하면 아래 함수를 실행해"라는 뜻이에요.

render_template('index.html')은 templates/ 폴더 안의 index.html 파일을 브라우저에 보내줘요.
편의점에 들어가면 직원이 "어서오세요!"라고 하듯이, 브라우저가 주소를 입력하면 서버가 HTML 페이지를 내보내는 거예요.

6 추천 라우트 — POST /recommend (핵심!)

@app.route('/recommend', methods=['POST']) def recommend(): data = request.get_json() user_input = data.get('keyword', '').strip() gender = data.get('gender', '성별 선택') age = data.get('age', '연령대 선택') skin_type = data.get('skin_type', '피부 타입 선택')
브라우저에서 "추천 받기" 버튼을 누르면 이 함수가 실행돼요.
request.get_json()으로 브라우저가 보낸 데이터(키워드, 성별, 연령대, 피부 타입)를 꺼내요.
.get('keyword', '')에서 두 번째 값 ''는 키워드가 없을 때 기본값(빈 문자열)이에요.

📍 경우 1: 키워드 없이 조건만 선택한 경우

if not user_input: filtered = df_skin.copy() if gender != '성별 선택': filtered = filtered[filtered['Gender'] == gender] if age != '연령대 선택': filtered = filtered[filtered['Age'] == age] if skin_type != '피부 타입 선택': filtered = filtered[filtered['Skin Type'] == skin_type] if filtered.empty: return jsonify({'error': '조건에 맞는 데이터가 없습니다.'}), 404 row = filtered.iloc[0] return _build_response(row, gender, age, skin_type)
키워드(피부 고민 문장)를 입력하지 않았을 때는, 선택한 조건(성별/연령/피부타입)에 맞는 데이터를 필터링해서 첫 번째 행을 돌려줘요.
filtered.empty는 조건에 맞는 데이터가 하나도 없을 때 True가 돼요. 이 경우 404 오류를 반환해요.
iloc[0]은 표에서 0번째(첫 번째) 행을 가져오라는 뜻이에요.

📍 경우 2: 키워드가 있는 경우 (AI 유사도 계산)

cleaned = preprocess_input(user_input) if not cleaned: return jsonify({'error': '유효한 단어를 찾을 수 없습니다.'}), 400 # 전체 데이터 TF-IDF 유사도 계산 input_vec = tfidf.transform([cleaned]) cosine_sim = linear_kernel(input_vec, tfidf_matrix)[0] # 프로필 일치 보너스 (항목당 0.05, 최대 0.15) profile_bonus = np.zeros(len(df_skin)) if gender != '성별 선택': profile_bonus += (df_skin['Gender'] == gender).values * 0.05 if age != '연령대 선택': profile_bonus += (df_skin['Age'] == age).values * 0.05 if skin_type != '피부 타입 선택': profile_bonus += (df_skin['Skin Type'] == skin_type).values * 0.05 best_idx = int((cosine_sim + profile_bonus).argmax()) row = df_skin.iloc[best_idx] return _build_response(row, gender, age, skin_type)
tfidf.transform([cleaned])
전처리된 사용자 입력 문장을 숫자 벡터(점수 배열)로 바꿔요. 예를 들어 "검버섯 생기다"가 [0.0, 0.8, 0.0, 0.4, ...] 같은 숫자 배열이 돼요.
linear_kernel(input_vec, tfidf_matrix)[0]
사용자 입력 벡터와 9031개 데이터 벡터를 하나씩 비교해서 유사도 점수(0~1)를 9031개 계산해요. 1에 가까울수록 비슷한 내용이에요.
코사인 유사도는 두 화살표가 얼마나 같은 방향을 가리키는지 보는 거예요. "검버섯"이라는 단어가 데이터 행에 많이 있을수록 같은 방향(높은 점수)이 돼요.
profile_bonus = np.zeros(len(df_skin))
9031개짜리 0으로 가득 찬 배열을 만들어요. 여기에 프로필이 일치하는 행마다 0.05씩 더해줘요. 성별·연령·피부타입 3개 모두 맞으면 최대 +0.15를 받아요.
왜 작은 보너스(0.05)만 주냐고요? 보너스가 너무 크면 피부 고민과 전혀 관계없는 내용이지만 프로필만 맞는 데이터가 1등이 되는 문제가 생겨요. 작은 보너스는 "유사도가 비슷할 때 프로필이 맞는 걸 조금 더 우선시"하는 역할만 해요.
(cosine_sim + profile_bonus).argmax()
유사도 점수 + 보너스 점수를 더한 최종 점수 배열에서 가장 큰 값의 위치(행 번호)를 찾아요. 그 행이 가장 추천할 만한 데이터예요.

7 응답 조립 함수 — _build_response()

def _build_response(row, gender, age, skin_type): profile_parts = [v for v in [gender, age, skin_type] if v not in ('성별 선택', '연령대 선택', '피부 타입 선택')] return jsonify({ 'profile': ', '.join(profile_parts) if profile_parts else '전체', 'response': row['Makeup Response'], 'rec_ingredients': row['Recommended Ingredients'], 'avoid_ingredients': row['Ingredients to Avoid'], })
profile_parts: 사용자가 선택한 값 중 "선택 안 함" 상태가 아닌 것들만 모아요. 예를 들어 성별만 "여성"을 선택했다면 ['여성']이 돼요.

jsonify({...}): 결과를 JSON 형식으로 브라우저에 보내요. JSON은 데이터를 주고받을 때 쓰는 표준 형식으로, 마치 택배 상자처럼 여러 정보를 담아서 보낼 수 있어요.
JSON 키내용예시
profile사용자가 선택한 프로필"여성, 50대 이상, 복합성"
response추천 솔루션 텍스트"검버섯 완화를 위해..."
rec_ingredients추천 성분"나이아신아마이드, 비타민C"
avoid_ingredients주의 성분"알코올, 향료"

8 서버 실행

if __name__ == '__main__': app.run(debug=True, port=5000)
if __name__ == '__main__': 이 파일을 직접 실행할 때만 서버를 시작해요. 다른 파일에서 import해서 쓸 때는 자동으로 서버가 켜지지 않아요.

debug=True: 개발 중에 코드를 수정하면 서버가 자동으로 재시작돼요. 실제 서비스를 배포할 때는 False로 바꿔야 해요.

port=5000: 서버가 5000번 포트를 사용해요. 그래서 주소가 http://127.0.0.1:5000 이에요.
포트 번호는 같은 컴퓨터 안에 있는 "방 번호"예요. 5000번 방에서 이 서버가 손님을 기다리는 거예요. 브라우저는 5000번 방으로 요청을 보내요.
